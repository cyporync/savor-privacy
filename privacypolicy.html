Savor Privacy Policy

Last updated: 11/29/2025

Contact: Savor Privacy & Security Team — savorprivacy@gmail.com

1. Summary

Savor is a mobile cookbook app that lets users save recipes (name, description, ingredients, materials) and optionally add a photo of the finished dish. This policy explains what data we collect, why we collect it, how we store it, and your choices.

2. Data we collect

  • Account data: username and hashed password (stored on our server) — used for authentication.
  • Recipe metadata: recipe name, description, ingredients, materials — stored in our MySQL database.
  • Recipe photos: if you add a photo, the app sends an encoded image to our PHP endpoint. The server decodes and saves the image as a .jpg file in the server uploads folder and stores the image URL in the database.
  • Local cache: minimal user info (username, user id) and a cached copy of recipe metadata may be stored on the device using SharedPreferences to speed up UI and provide limited offline access. Image files are not persisted in the app beyond standard media URIs and view cache unless explicitly saved by the user.
  • Device & permission info: camera permission is requested only when you use the “Take Photo” feature. We do not collect location, microphone, or any other sensor information.

3. How we use the data

  • Authenticate and authorize access to a user’s recipes.
  • Store and serve recipes and associated images so you can view and edit them across devices.
  • Cache small amounts of data locally to improve performance and provide limited offline access.

4. How we share and with whom

We do not sell or share your personal data with third parties for marketing. Recipes and images are stored on our server hosted by Michigan State University College of Engineering DECS (Division of Engineering Computing Services).

5. Third-party services and libraries

We use the following third-party libraries in the app:

  • Retrofit2 (network communication with backend)
  • OkHttp (logging interceptor and network debugging)
  • Guava (utility library used by AndroidX)
  • Firebase Crashlytics (crash reporting)
  • Firebase In-App Messaging (in-app message support)
  • CameraX (camera preview)
  • Gson (JSON parsing)
  • Glide (loading and displaying images)
  • JUnit/Espresso/AndroidX Test (testing libraries)

These libraries may transmit or log request/response data for debugging. We do not use advertising SDKs in the current build.

6. Storage, retention, and deletion

  • Recipe metadata and image URLs are stored permanently on the server until you delete the recipe.
  • When you delete a recipe in the app, the server removes the database record and attempts to delete the corresponding image file from the server uploads folder.
  • If you want your data removed entirely, contact us at savorprivacy@gmail.com with your username. We will delete your account and associated recipes on request.

7. Security

Passwords are hashed on the server. Communication between the app and server uses HTTPS. Access to the server is limited to the team. While we take reasonable precautions to protect data, no system is completely secure. By using the app you accept this small residual risk.

8. Permissions

The app requests only the permissions necessary for functionality: camera permission when using the “Take Photo” feature, permission to write to external storage to save recipe photos, and network access. Each permission is requested at runtime and explained in the UI before being requested.

9. Children

The app is not intended for children under 13. We do not knowingly collect data from children. If you believe a child’s data is present, contact us so we can remove it.

10. Changes to this policy

We may update this policy to reflect changes in features or legal requirements. We will post the revised policy at this URL and update the “Last updated” date.

11. Contact

If you have privacy questions or want your data deleted, contact: savorprivacy@gmail.com.
